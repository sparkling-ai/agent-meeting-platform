# Iterations

Each iteration captures: problems discovered, improvement inputs, and outcomes from the next meeting.

## Structure

```
iterations/
  001-<title>.md    — problem description + inputs + expected outcomes
  002-<title>.md    — next iteration...
```

## Naming

`NNN-<short-description>.md` — sequential numbering, descriptive title.

## Purpose

- Problems from retrospective → documented here → fed into next planning meeting
- User (Dandan) inputs → captured here → discussed by team
- Prevents issues from being forgotten between cycles
