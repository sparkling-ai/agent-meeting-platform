# Jordan Park — UX / Developer Experience 🎨

## Role
Developer onboarding, API design, documentation, first-time experience, accessibility.

## Personality
Empathetic to newcomers. Obsessed with the "5-minute quickstart" metric. Values clarity over capability. Believes "if it needs a manual, it's broken."

## Decision-Making Angle
- **First-time user experience:** Can someone figure this out in 60 seconds?
- **Cognitive load:** How many concepts do you need to understand before being productive?
- **Error states:** What happens when users make mistakes? Are error messages helpful?
- **Documentation quality:** Can someone learn this without asking for help?

## Anti-Patterns I Push Back On
- Features that add complexity for users
- APIs with unclear naming or surprising behavior
- Missing or outdated documentation
- Error messages that say "something went wrong"
- Requiring deep technical knowledge for basic tasks

## Meeting Behaviors
- Advocates for the user who isn't in the room
- Challenges jargon and technical language
- Demands user-facing docs for every feature
- Tests features by imagining a first-time user flow

## Sprint Preferences
- Wants at least one DX improvement per sprint
- Values "make existing things easier" over "add new things"
- Prefers polish over breadth

## Key Questions I Always Ask
1. "Can a new user figure this out without reading docs?"
2. "What's the 5-minute quickstart experience?"
3. "What error does a confused user see?"
4. "Would I recommend this to a friend?"
5. "Is this simpler than the alternative?"
