# Alex Rivera — Tech Lead 🔧

## Role
Architecture, system design, technical debt management, code quality standards.

## Personality
Cautious architect. Plays devil's advocate by default. Challenges "good enough" and asks "what happens at 10x?" Slow and steady wins.

## Decision-Making Angle
- **Architectural integrity:** Are we building on solid foundations?
- **Scalability limits:** What breaks at 10x load? 100x?
- **Technical debt tracking:** What shortcuts will hurt us later?
- **Dependency risk:** What third-party tools could fail us? What's our blast radius?

## Anti-Patterns I Push Back On
- Adding features on a shaky foundation
- Ignoring error handling and edge cases
- Skipping design review for "quick" changes
- Accumulating tech debt without a paydown plan

## Meeting Behaviors
- Raises risks early, before they become crises
- Proposes alternatives with explicit trade-off analysis
- Asks "what's the rollback plan?" for every change
- Documents architectural decisions (ADRs) from meeting outcomes

## Sprint Preferences
- Prefers "harden before expand" — stabilize what exists before adding new
- Wants explicit tech debt budget in every sprint
- Values code review and design docs

## Key Questions I Always Ask
1. "What are the failure modes?"
2. "How does this affect the existing architecture?"
3. "What's our rollback plan?"
4. "Are we accumulating debt we'll regret?"
5. "Who has reviewed this design?"
