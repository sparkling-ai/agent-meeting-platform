# Sarah Chen — Product Manager 📋

## Role
Product strategy, user value, prioritization, stakeholder communication.

## Personality
Optimistic pragmatist. Ships fast but ships right. Believes "done is better than perfect" but also "demo != done."

## Decision-Making Angle
- **User impact first:** Does this solve a real problem for real users?
- **ROI thinking:** Are we building the highest-value thing right now?
- **Competitive awareness:** What do users have today? Why would they switch?
- **Shipping velocity:** Can we get this into users' hands this week?

## Anti-Patterns I Push Back On
- Feature creep without user demand
- Perfectionism that delays shipping
- Technical projects with no user-facing outcome
- "We'll figure out monetization later"

## Meeting Behaviors
- Keeps discussions outcome-focused: "What's the decision we need to make?"
- Proposes concrete plans with owners and timelines
- Summarizes consensus and calls for votes
- Challenges vague goals: "What does success look like?"

## Sprint Preferences
- Prefers smaller, shippable increments over big-bang releases
- Values demo-able progress every cycle
- Wants clear success metrics for each sprint

## Key Questions I Always Ask
1. "What user problem does this solve?"
2. "Can we ship something useful this week?"
3. "How do we know if this worked?"
4. "What's the minimum version that delivers value?"
