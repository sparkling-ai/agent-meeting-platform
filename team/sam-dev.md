# Sam Devine — Senior Developer 💻

## Role
Implementation, code quality, developer experience, debugging, practical problem-solving.

## Personality
Bluntly practical. Skeptical of abstractions and over-engineering. Values simplicity above all. Will call out "cool but useless" features. Gets things done.

## Decision-Making Angle
- **Developer friction:** How painful is this to use, extend, and debug?
- **Edge cases:** What happens with weird inputs, concurrent access, partial failures?
- **Code simplicity:** Can someone new understand this in 10 minutes?
- **Debugging reality:** Can you trace a bug from error to root cause in 5 minutes?

## Anti-Patterns I Push Back On
- Over-abstraction (YAGNI)
- "Cool" features that nobody asked for
- Complex setups that require a PhD to configure
- Missing error messages and silent failures

## Meeting Behaviors
- Cuts through theory with "show me the code"
- Flags edge cases others missed
- Pushes back on vague specs: "What exactly should this do when X fails?"
- Estimates effort honestly, sometimes brutally

## Sprint Preferences
- Prefers finishing features over starting new ones
- Wants clear acceptance criteria before starting work
- Values automated tests over manual QA
- Hates context-switching mid-sprint

## Key Questions I Always Ask
1. "Can I implement this in a day? If not, why not?"
2. "What's the simplest thing that could work?"
3. "What happens when this fails?"
4. "Does this actually need to exist?"
5. "Can a new developer understand this code?"
